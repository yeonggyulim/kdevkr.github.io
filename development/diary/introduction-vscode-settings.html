잠만보가 사용하는 VSCode 설정

  • DEVELOPMENT
  • DIARY
  • INTRODUCTION-VSCODE-SETTINGS

2018년은 아톰 이라는 에디터를 사용해왔다.

개인적으로 아톰이 유저 인터페이스도 직관적이고 프로젝트 폴더 관리가 유용하다고 생각한다.

하지만, 개발 시 유용한 다양한 플러그인을 추가하다보면 점점 느려지는 아주 커다란 문제점을 가지고 있다.


아톰을 쓰다보면 점점 느려진다

2018년 1월 10일에 아톰 공식 블로그에도 성능에 관한 포스트인 The State of Atom’s Performance이 작성되었다.

그러나 많은 업데이트에도 불구하고 플러그인이 충돌되는 현상과 느려지는 문제점은 개선되지 않았다.

그래서 2019년에는 VSCode를 써보고자 한다.

기존 아톰에서 사용했던 설정을 최대한 그대로 사용할 것이다.

유저 인터페이스 꾸미기

테마

잠만보가 사용했던 테마 조합은 아톰의 기본값인 One Dark였다.

  • Atom One Dark Theme
  • One Dark Pro ⭐

VSCode의 확장 플러그인 중에서 비슷하고 깔끔한 One Dark Pro를 사용한다.

아이콘

file-icons도 있지만 vscode-icons를 사용한다.

개발 환경 구성하기

개발 시 활용되는 패키지를 구성하고 관련된 설정을 해보자.

아톰 단축키 적용

Atom에서 사용하던 단축키를 VSCode에서 사용할 수 있도록 지원하는 Atom Keymap을 설치하자.

구문 하이라이팅

Vue

Vue를 위한 구문 하이라이팅 확장 플러그인으로 Vetur를 설치한다.

구문 하이라이팅 뿐만 아니라 다양한 기능(스니핏, 구문 분석)을 제공한다.

Formatting

포맷을 위한 확장 플러그인으로 prettier를 선택했다.

구문 하이라이팅을 위해 설치한 Vetur에서 Formatter로 prettier를 사용할 수 있다.

코드 분석

코드 분석을 위해 ESLint 확장 플러그인을 설치한다.

구문 하이라이팅을 위해 설치한 Vetur에서 Linter로 ESLint를 사용할 수 있다.

유용하게 사용할 수 있는 확장 플러그인

VSCode를 좀더 유용하게 사용할 수 있는 확장 플러그인을 추천한다.

Auto Close Tag & Auto Rename Tag

Auto Close Tag와 Auto Rename Tag는 HTML이나 XML 태그 작성 시 유용하다.


Bracket Pair Colorizer

Bracket Pair Colorizer은 브라켓이 중첩될 때 색깔별로 표시하여주는 확장 플러그인이다.

IntelliSense for CSS class names in HTML

IntelliSense for CSS class names in HTML는 워크스페이스에 존재하는 CSS 클래스를 찾아 알려준다.

npm Intellisense

npm Intellisense는 Import 구문 시 노드 패키지 모듈을 추천해준다.

Path Intellisense

Path Intellisense는 경로 기반으로 파일 이름을 찾아 추천해준다.

SCSS IntelliSense

SCSS IntelliSense는 SCSS 관련 변수 또는 믹스인 등을 추천해준다.

vscode-pigments

vscode-pigments는 색상 코드값을 컬러로 표현해준다.

Debugger for Chrome

Debugger for Chrome는 크롬 브라우저에서의 디버깅을 VSCode에서 할 수 있게 해준다.

VSCode Browser Sync

VSCode Browser Sync는 Browser Sync 기능을 통한 실시간 리로딩을 제공한다.

실시간 페어 프로그래밍

VS Live Share은 워크스페이스를 팀원끼리 공유할 수 있게 한다.

VSCode 동기화 하기

아톰에 sync-settings이 있다면 VSCode에서도 Settings Sync로 설정을 동기화 할 수 있다.
